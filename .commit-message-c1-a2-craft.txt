C1-A2 — check_no_dropped_stages (src/craft/run_record.py):
- New public callable check_no_dropped_stages(canonical, archived_runs)
  -> list[str] (+ _completed_stage_ids helper). A resume must NEVER lose a
  stage a prior run completed: the canonical's `completed`-stage-id set
  MUST be a superset of every archived runs/run-N/run_record.json's
  `completed` set. Returns error strings naming any dropped stage; empty =
  complete. Manifest-free — compares only against what runs actually
  completed. This is the invariant the per-record validator CANNOT see
  (its reconciliation only checks totals==sum(stages present), so an
  incomplete record reconciles perfectly — the exact C1-A drop class).
- Deliberately NOT folded into validate_run_record: that's a single-record
  schema validator; this is a cross-record comparison and stays a separate
  callable the finalize path drives. Skips the canonical's own
  same-run_id archive snapshot (it can't drop its own stages). Tolerant of
  malformed inputs; never raises.
- The skills VENDOR a copy (they ship standalone on the hub with no craft
  on PYTHONPATH — same copy-not-share constraint as the run-record
  validator). Family-H (below) pins the copies behaviorally.

Family-H (tests/test_conformance.py) — behavioral identity:
