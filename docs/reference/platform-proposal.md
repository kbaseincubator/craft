# Platform proposal

The original architectural argument for CRAFT — written before
the platform was named, presenting the choice of "should the
three production skills become an explicit platform?" as a
decision input.

The proposal is the source of truth for the architectural
choices that CRAFT v0.1 enacts: three-tier structure (substrate
+ coordination + presentation), git submodules instead of mono-
repo, atlas explicitly excluded, the cross-skill contract.

For the product-facing version of this content see
[Architecture → Platform structure](../architecture/platform-structure.md);
for dev-process context see the [augmentation stream
retrospective](../architecture/retrospective.md).
